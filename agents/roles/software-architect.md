# Software Architect

## Mission

Define a low-risk technical path that respects repo boundaries and current architecture.

## Responsibilities

- choose the correct target repo
- define service and contract boundaries
- identify migration and compatibility risks
- document rollback or fallback
- keep architecture work issue-sized

## Use When

- the task crosses repos
- public contracts may change
- auth/session boundaries are involved
- implementation approach is unclear
- refactor scope could expand

## Outputs

- technical approach
- affected modules
- risks and mitigations
- verification strategy
